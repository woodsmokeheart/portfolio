## 1. Цель и контекст

Личный сайт-портфолио QA Lead. Главная задача — **личный бренд и узнаваемость в QA-сообществе**. Сайт служит «якорем»: люди приходят из ВК, статей, LinkedIn — и видят выверенную страницу вместо случайных профилей.

Ключевые требования:
- Двуязычность (RU / EN) с **явным** переключателем, видимым пользователю.
- Mobile-first: отлично выглядит и работает на телефонах.
- Возможность временами публиковать статьи **без бэкенда**.
- Визуально **необычный и не скучный** — премиальный минимал (Linear/Apple-уровень) + wow-эффекты.
- Отдельная самодостаточная папка, которую можно вынести из `jessai/` без проблем.

## 2. Концепция: «Сайт как тикет»

Весь сайт оформлен как один большой QA-тикет про Denis. Метафора — остроумный слой сверху, **не барьер**: каждый раздел имеет и название поля тикета, и понятную человеческую подпись, чтобы пользователь не из контекста QA не потерялся.

Навигация: **единый вертикальный скролл** сверху вниз, как чтение тикета. Слева — закреплённая (sticky) панель меты тикета с быстрыми ссылками. На мобиле панель схлопывается в верхний sticky-бар + кнопку быстрой навигации по секциям.

### Маппинг полей тикета на разделы

| # | Поле (стилевой тег) | Человеческая подпись | Контент |
|---|---|---|---|
| 1 | Title | Кто я | Имя, большое фото, хедлайн, статус |
| 2 | Description | Обо мне | Короткий питч |
| 3 | Preconditions | Бэкграунд | Как пришёл в QA, контекст |
| 4 | Steps to Reproduce | Путь | Карьера нумерованными шагами (таймлайн) |
| 5 | Expected Result | Цели | К чему стремлюсь |
| 6 | Actual Result | Кейсы | Реальные достижения (карточки) |
| 7 | Environment | Стек | Скиллы + инструменты |
| 8 | Attachments | Статьи + контакты | Список MDX-статей, ссылки, почта |

## 3. Стек

Версии согласованы с соседним проектом `qa-sprint-board` для единообразия.

- **Next.js 16** (App Router, SSG/static export где возможно) + TypeScript
- **React 19**
- **Tailwind CSS v4** (`@tailwindcss/postcss`)
- **next-intl** — локализация RU/EN через сегмент `[locale]`
- **motion** (framer-motion) — анимации и scroll-triggered эффекты
- **MDX** для статей (`@next/mdx` + `gray-matter` для фронтматтера, либо `next-mdx-remote`)
- **lucide-react** — иконки (как в qa-sprint-board)
- **Vercel** — хостинг (free tier)

Решение по MDX: используем `@next/mdx` с файловым роутингом статей через `content/articles/*.mdx`, метаданные читаем `gray-matter`. Это исключает CMS и БД.

## 5. Левая панель (мета тикета)

Sticky на десктопе, схлопывается на мобиле. Содержит:
- Номер тикета: `QA-001`
- Статус: `OPEN` / `Available for work` — пульсирующий бейдж (зелёная точка с glow)
- Priority: `Highest`
- Assignee: фото-аватар + имя
- Labels: `qa-lead`, `automation`, `kafka` (стилизованные пилюли)
- Переключатель `RU / EN`
- Быстрые ссылки на все 8 секций; активная подсвечивается по мере скролла (scroll-spy)

## 6. Wow-эффекты (изюминка)

- **Glow за курсором** — мягкое акцентное радиальное свечение следует за мышью (desktop; на тач отключается).
- **Анимированные счётчики** — числовые метрики (21 микросервис, 5+ лет) считаются от нуля при появлении.
- **Scroll-triggered reveals** — секции и элементы появляются с fade + slide при попадании во вьюпорт (`whileInView`).
- **Stagger** — списки и карточки появляются поочерёдно.
- **Таймлайн рисуется** — линия пути (Steps to Reproduce) прорисовывается по мере скролла.
- **Полоски скиллов** — заполняются с ease-анимацией при появлении.
- **Pop-in тегов** — инструменты появляются с лёгким масштабированием.
- **Переходы статусов** — пульсация бейджа Available.

Все анимации уважают `prefers-reduced-motion` — при включённой настройке упрощаются/отключаются.

- Маленький аватар — в панели меты как assignee.

## 8. Статьи (MDX)

- Файл на язык `content/articles/<slug>.<locale>.mdx` с фронтматтером:
  ```yaml
  ---
  title_ru: "Заголовок"
  title_en: "Title"
  date: "2026-06-24"
  tags: ["qa", "kafka"]
  cover: "/articles/<slug>/cover.png"
  excerpt_ru: "..."
  excerpt_en: "..."
  ---
  ```
- Список статей рендерится в секции Attachments и на `/[locale]/articles`.
- Отдельная статья — `/[locale]/articles/[slug]`, MDX рендерится в стилизованный layout.
- Никакого бэкенда: при добавлении файла и пуше Vercel пересобирает статику.

## 9. Локализация

- `next-intl`, сегмент `[locale]` в роуте (`/ru`, `/en`).
- Переключатель сохраняет текущую секцию/статью при смене языка.
- Дефолтная локаль определяется по `Accept-Language`, с возможностью ручного переключения (cookie).
- Весь UI-текст — в `messages/ru.json` и `messages/en.json`.
- Контент статей: тело статьи — отдельный файл на язык `content/articles/<slug>.<locale>.mdx` (`<slug>.ru.mdx`, `<slug>.en.mdx`). Общие метаданные (дата, теги, cover) и двуязычные `title_*/excerpt_*` — во фронтматтере каждого файла. Статья показывается в списке, если существует хотя бы один языковой вариант; при отсутствии перевода переключатель ведёт на доступный язык с пометкой.

## 10. SEO

- `generateMetadata` на локаль и на статью (title, description, og:image).
- Статическая генерация (SSG) всех страниц.
- `sitemap.xml` и `robots.txt`.
- Семантический HTML, контент в DOM (не в canvas) — индексируется.

## 11. Визуальный язык

- Тёмная тема (основная). Светлая — НЕ в этой итерации (YAGNI).
- Типографика: системный sans-serif стек, крупные заголовки с отрицательным letter-spacing.
- Акцентный цвет — сдержанный (синий/teal из токенов), используется точечно.
- Моноширинные подписи для «тикетных» тегов (номер, статусы, поля).
- Никаких градиентов-слопа, теней-перебора; флэт и чисто.

## 12. Сознательно вне scope (YAGNI)
